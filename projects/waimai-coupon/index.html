Event-Based Funnel + Redemption Prediction

美团优惠券投放效果分析

本项目以每一次实际领券事件为样本，追踪“领券 → 下单 → 同券核销”的完整链路，识别当前券为何没有被真正使用，并通过核销预测模型排序高潜力用户-券组合，支持从泛发券转向精细化投放。

Data Scope & Cleaning

数据链路与口径治理

先区分可解释缺失与会污染当前券核销口径的异常，再构建可追踪的事件级样本。

从三张原始表到事件链路

  • 领券表：12,693,100 行，包含用户、券、领取日期、有效期、金额与门槛。
  • 订单表：1,048,575 行，包含订单、支付日期、实付金额、减免金额与用券信息。
  • 访问表：2,815,177 行，用于刻画用户近期需求强度与行为活跃度。
  • 清洗动作：SQL + Python 多表关联、链路校验、缺失值治理、日期/金额异常检查与右截断处理。

关键质量校验

重复值三张 cleaned tables 重复率均为 0%。
异常剔除无券 ID 但存在减免金额的订单不混入当前券核销分析。
链路可追踪订单用券记录按 User_id + Coupon_id 回溯领券表，匹配率约 91.96%。

Funnel Diagnosis

漏斗拆解与问题定位

断点不是用户完全不下单，而是下单后没有使用当前券；问题更像“券匹配效率不足”。

领券 → 下单并不低

62.8%

用户领取优惠券后仍有较高比例在窗口期内下单，说明需求并非完全缺失。

下单 → 同券核销极低

4.9%

大量订单没有使用本次领取的券，导致补贴触达与实际核销脱节。

其他券替代是主因

78.6%

下单但未核销当前券的事件中，大部分被其他券替代。

业务问题重述

平台需要判断哪张券在什么场景下真正匹配用户需求，而不是继续扩大粗放发券规模。

Modeling

事件级核销预测模型

预测“哪一次领券事件更可能在 7 天内转化为当前券核销”，用于投放排序。

标签、特征与训练

  • 样本：User_id + Coupon_id + Receive_date；主标签为 7 天内同券核销。
  • 样本结构：建模样本约 1.30M，正样本 40,642，正样本比例 3.13%。
  • 特征工程：围绕用户需求、券吸引力、门槛匹配、多券竞争构建 45 个数值特征。
  • 训练方案：Logistic Regression 作为基线，HistGradientBoosting 捕捉非线性与交互；通过分层抽样、时间切分和类别不平衡处理控制评估偏差。

模型表现

PR-AUC0.344
Top10% Precision20.95%，Lift 6.38×
Top1% Lift17.9×
解释结果券有效期、折扣力度、门槛匹配度对核销影响最大。

Business Strategy

业务策略与落地价值

把模型分数转化为可执行的投放优先级和券设计建议。

谁该发

优先触达近期有需求、历史用券倾向较高且模型预测分数高的事件；对高自然转化但低核销概率事件减少补贴。

发什么券

重点优化券有效期、折扣力度与门槛匹配度，减少被更优券替代。

什么时候发

在用户刚出现需求信号、下单意愿尚未完全自然转化且多券竞争不拥挤时触发。

一句话总结

优惠券运营的关键不是“发出去”，而是让合适的券在合适时间被合适的人真正用掉。
